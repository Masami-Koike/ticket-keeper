# TicketKeeper 詳細仕様書

ミスドの引換券・株主優待・クーポン・ギフト券などの有効期限を一元管理する、単一HTMLのクライアントサイドアプリ。
本書はデータモデル・期限判定ロジック・UI挙動・入出力仕様をまとめたもの。

---

## 1. 目的と前提

- バラバラに保有しているチケット・優待類を 1 か所にまとめ、**期限切れによる失効を防ぐ**。
- サーバー・ログイン・外部通信なし。全データは閲覧端末のブラウザ内 (`localStorage`) に保存。
- GitHub Pages 等の静的ホスティングにそのまま設置できる。

---

## 2. データモデル

`localStorage` のキー `ticketkeeper_v1` に、チケット配列を JSON 文字列で保存する。

各チケットのオブジェクト:

| フィールド | 型 | 必須 | 説明 |
|-----------|------|------|------|
| `id`     | string  | ○ | 一意な識別子。`Date.now()`(36進) + 乱数5桁で生成 |
| `name`   | string  | ○ | チケット名(例: ミスド ドーナツ引換券) |
| `cat`    | string  | ○ | カテゴリ。後述の固定 6 種のいずれか |
| `qty`    | number  | − | 枚数・数量。0 のときは非表示(整数、既定 0) |
| `expiry` | string  | ○ | 有効期限。`YYYY-MM-DD` 形式 |
| `note`   | string  | − | メモ(店舗・金額・利用条件など) |
| `used`   | boolean | ○ | 使用済みフラグ |
| `added`  | number  | ○ | 登録日時(`Date.now()` のミリ秒) |

### カテゴリ (固定)

`ミスド` / `株主優待` / `クーポン` / `ギフト券` / `ポイント` / `その他`

---

## 3. 期限判定ロジック

基準日は「今日(ローカル時刻の 0:00)」。`daysUntil = floor((期限0:00 − 今日0:00) / 1日)` を四捨五入で算出。

| ステータス | 条件 | 色(左ボーダー/残日数) |
|-----------|------|----------------------|
| `used`    | `used === true`(最優先) | グレー(半透明・打ち消し線) |
| `expired` | 残日数 < 0 | 赤 |
| `soon`    | 0 ≤ 残日数 ≤ 7 | 橙 |
| `month`   | 8 ≤ 残日数 ≤ 30 | 黄 |
| `ok`      | 残日数 > 30 | 既定(無色) |

残日数の表示文言: `期限切れ`(<0) / `本日まで`(=0) / `あと N 日`(>0) / `使用済`(used)。

### サマリ

画面上部に常時表示。`used` を除いたアクティブ分について集計:
- 登録総数(全件)
- 7 日以内(0 ≤ 残日数 ≤ 7)の件数 — 橙ピル
- 期限切れ(残日数 < 0)の件数 — 赤ピル

---

## 4. 一覧表示

### 絞り込み(AND 条件)

1. **カテゴリ**: チップで選択(`すべて` または特定カテゴリ)。
2. **使用済み・期限切れの表示**: チェックボックス。OFF のとき `used` および残日数 < 0 を除外。
3. **検索**: `name + note` に対する部分一致(小文字化して比較)。

### 並び替え

| 値 | 内容 |
|----|------|
| `expiry`(既定) | 期限が近い順(昇順) |
| `expiry-desc`  | 期限が遠い順(降順) |
| `name`         | 名前順(`localeCompare` ja) |
| `added`        | 登録が新しい順(`added` 降順) |

## 5. 操作仕様

### 追加 / 編集

- 右下 **＋** で追加モーダル、各行 **✎ 編集** で編集モーダルを開く。
- 必須: `name`, `expiry`。未入力時は保存しない。
- `qty` は整数パース、不正値は 0。
- 編集時は `id` で対象を特定し `Object.assign` で上書き(`id`/`added`/`used` は保持)。

### 使用済みトグル / 削除

- **✓ 使用済 / ↩ 戻す** で `used` を反転。
- **🗑 削除** は確認ダイアログの後、`id` 一致を配列から除去。

### モーダルの閉じ方

- キャンセルボタン / オーバーレイ余白クリック / `Esc` キー。

---

## 6. 入出力(バックアップ)

右上メニュー(`⋯`)から操作。

### 書き出し

`items` を `JSON.stringify(_, null, 2)` で整形し、`ticketkeeper-backup-YYYY-MM-DD.json` としてダウンロード。

### 読み込み

- JSON ファイルを選択して読み込む。配列でなければエラー。
- 既存データがある場合、`OK = 現在のデータにマージ` / `キャンセル = 全置き換え` を選択。
- マージ時は `id` 重複を除外して追加。
- 各レコードは欠損フィールドを既定値で補完して正規化。

### 全データ削除

確認ダイアログの後、`items` を空にして保存(事前バックアップを推奨表示)。

---

## 7. UI / 表示

- ダークテーマ、最大幅 640px のモバイルファースト。アクセントカラー `#ff6b35`。
- PWA メタ(`apple-mobile-web-app-capable` 等)対応。iOS の「ホーム画面に追加」でフルスクリーン起動。
- ファビコン/アイコンはインライン SVG(チケット型)。
- 依存ライブラリなし。すべて 1 ファイル(`index.html`)に内包。

---

## 8. 既知の制約 / 今後の拡張余地

- 複数端末間の自動同期は非対応(設計上、静的ホスティング前提)。GitHub への JSON 保存方式や、
  ブラウザ通知による期限アラートは将来拡張の候補。
- カテゴリは現状固定。ユーザー定義カテゴリの追加は今後の検討事項。
- `localStorage` 依存のため、ブラウザのサイトデータ消去でデータが失われる(バックアップで担保)。
