# TicketKeeper

ミスドの引換券・株主優待・クーポン・ギフト券などの**有効期限をまとめて管理**するアプリ。
バラバラに持っていて期限を忘れがちなチケット類を 1 か所に集め、期限が近い順に並べて色で警告します。

ブラウザで `index.html` を開くだけで動作し、サーバーもインストール作業も不要です。

---

## 特徴

- **期限が近い順に自動ソート＋色分け** — 🔴 期限切れ / 🟠 7 日以内 / 🟡 30 日以内 で一目で危険度がわかる
- **危険度サマリ** — 「7 日以内 ○件」「期限切れ ○件」を画面上部に常時表示
- **カテゴリ管理** — ミスド / 株主優待 / クーポン / ギフト券 / ポイント / その他 で絞り込み
- **検索・使用済み管理** — 名前/メモで検索、使った券は「使用済」にして残せる(削除しない)
- **単一HTMLファイル** — 依存ライブラリなし、`index.html` だけで完結
- **オフライン動作** — 全処理クライアントサイド、`localStorage` で状態保存
- **モバイルファースト** — スマートフォンでの利用を最適化、iOSはホーム画面追加にも対応
- **バックアップ書出/読込** — JSON で書き出し・読み込みでき、機種変更や別端末への移行に対応

## 使い方

### セットアップ

```bash
git clone https://github.com/Masami-Koike/ticket-keeper.git
cd ticket-keeper
```

`index.html` をブラウザで開くだけで起動します。

```bash
# macOS
open index.html
# Windows
start index.html
# Linux
xdg-open index.html
```

### GitHub Pages で公開する場合

リポジトリの **Settings → Pages → Source** で `main` ブランチのルートを指定すれば、そのまま公開できます。
公開後はスマホでURLを開き、ブラウザの「ホーム画面に追加」をするとアプリのように使えます。

### 基本フロー

1. 右下の **＋** ボタンからチケットを登録(名前・カテゴリ・枚数・有効期限・メモ)。
2. 一覧は**期限が近い順**に並び、期限切れ=赤 / 7日以内=橙 / 30日以内=黄で色分け表示。
3. 使ったチケットは **✓ 使用済** に切り替え(削除せず履歴として残せる)。
4. 上部のカテゴリチップ・検索ボックスで絞り込み。
5. 右上メニュー(`⋯`)から **バックアップの書き出し/読み込み**、全データ削除が可能。

## データの保存について

- データは**開いた端末・ブラウザの中だけ**に `localStorage` で保存され、サーバーには送信されません。
- 別の端末・ブラウザには自動同期されません。移行したいときはメニューから
  「⬇ バックアップを書き出す」で JSON を保存し、移行先で「⬆ バックアップを読み込む」してください。
- ブラウザのサイトデータを消去すると登録内容も消えます。大事なデータは定期的にバックアップを推奨します。
- 読み込み時は「現在のデータに追加(マージ)」と「全て置き換え」を選べます(同一 ID は重複登録されません)。
